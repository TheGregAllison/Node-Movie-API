MyFlix API v1.0.0

The MyFlix API is built with the REST architecture in mind. This API has self-explanatory URLs, returns JSON responses, and uses standard HTTP response codes.

Errors are returned as standard HTTP response codes. Messages detailing the error are included in the response body.


Supported Requests

Request URL HTTP Method Authorization Request body Response body
User Login /login POST None A JSON object holding Username and Password, structured as follows:

{ 'Username': '{username}', 'Password': '{Password}' },
 JWT Authentication token
{ 'token': '######' }
Register a new user /users POST None A JSON object holding data about the user to add, structured like:

{ 'Name': 'John Doe',
'Username': '{username}',
'Email': {email@example.com},
'Birth-date': 'YYYY-MM-DD' } 		A JSON object holding data about the user that was added, including their assigned id and auth token:
{ '_id': '######',
'Name': 'John Doe',
'Username': '{username}',
'Email': {email@example.com},
'Birth-date': 'YYYY-MM-DD',
'token': '######' }
Get list of movies /movies GET Required (JWT) None An array of JSON objects holding data about all the movies in the database
Get list of movies by director /movies/directors/{Name} GET Required (JWT) None An array of JSON objects holding data about all the movies by requested director
Get list of movies by genre /movies/genres/{genreName} GET Required (JWT) None An array of JSON objects holding data about all the movies by requested genre
Get data about a single movie by title /movies/{Title} GET Required (JWT) None A JSON object holding data about the requested movie like this:
{ '_id': '######',
'title': 'Iron Man',
'director': 'John Favreau',
'release-year': '2008',
'genre': 'action' }
Update user details /users/{Username} PUT Required (JWT) A JSON object holding data about the user to update, structured as follows:

{ 'Name': 'John Doe',
'Username': '{username}',
'Email': {email@example.com},
'BirthDate': 'YYYY-MM-DD' } 		A JSON object holding data about the user that was added, including their assigned id and auth token:
{ '_id': '######',
'Name': 'John Doe',
'Username': '{username}',
'Email': {email@example.com},
'BirthDate': 'YYYY-MM-DD',

Delete User /users/{Username} DELETE Required (JWT) None A confirmation stating the user has been deleted
Add a movie object to a user's favorites list /users/{Username}/movies/{movieID} PUT Required (JWT) None A text message indicating whether the movie was successfully added from the user's favorited movies
Remove a movie from a user's favorites list /users/{Username}/movies/{movieID} DELETE Required (JWT) None A text message indicating whether the movie was successfully removed from the user's favorited movies
